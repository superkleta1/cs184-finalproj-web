CS184/284A Spring 2025 Homework 4 Write-Up

Non-photorealistic Shader in Minecraft

Zaychik Liu, Candice Yang, Annika Liu, Andy Zhang

Link to webpage: superkleta1.github.io/cs184-finalproj-web/

Summary

In our project, we aim to create some non-photorealistic shaders in Minecraft, using Minecraft’s given rendering pipeline. Some shader effects we want to try creating are cel-shading, painterly-shading, and watercolor-shading.

Problem Description

Minecraft is a sandbox game with ample support for user-generated content and customization. In particular, it supports user-created shaders and provides a well-structured rendering pipeline allowing for flexible integration. We would like to apply what we learned from CS184’s raytracing, shading, and illumination module to create a few non-photorealistic shaders in Minecraft. Some technical challenges are how to represent non-photorealistic art styles using math and codes.

Goals and Deliverables

What We Plan to Deliver

At the basis, we would like to complete a cel-shading shader for Minecraft. After the shader is applied, we expect the game to look cartoonish, with close-to-none smooth shadows, strong color contrast, and thick outlines. We hope to achieve an effect similar to this. The result of our system can be easily measured by a comparison of the in-game effects between vanilla shader and our shader.

What We Hope to Deliver

Besides implementing a cel-shader, we also hope to implement a painterly-shader and a water-color shader. We hope to achieve effects similar to the following. Whether we get to implement these depends on the progress on our cel-shader and time available.

Schedule

Week 1 - Learn Minecraft rendering pipeline, practice using OpenGL to write Minecraft custom shaders

Week 2 - Start implementation of cel-shading

Week 3 - Finish implementing cel-shading

Week 4 - Implement painterly-shader and water-color shader
